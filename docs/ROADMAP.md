# Grandpa's New Ears 项目路线图

## Phase 0：产品定义

- 建立初版 PRD。
- 固定 MVP 范围。
- 明确试点用户类型。
- 决定第一版平台：移动端网页 / PWA 或原生 App。
- 制作低保真实时字幕交互原型。

## Phase 1：技术验证

- 测试主流移动浏览器的麦克风采集能力。
- 比较可用的流式语音识别方案。
- 测量临时字幕延迟。
- 验证基础翻译流程。
- 确认隐私、数据保留和成本边界。

## Phase 2：MVP 开发

- 开发实时字幕屏。
- 增加语言和翻译设置。
- 增加字号、高对比度等无障碍控制。
- 增加保存或丢弃对话流程。
- 增加已保存对话历史。
- 开发基础后端网关。

## Phase 3：用户试点

- 与听障人士、老人和照护者进行可用性测试。
- 收集可读性、延迟、准确率反馈。
- 调整默认字号、对比度和字幕位置。
- 改进错误恢复和首次使用流程。

## Phase 4：产品扩展

- 评估原生 iOS 和 Android 应用。
- 探索离线或混合语音识别。
- 增加自定义词表。
- 为保存的对话增加摘要。
- 增加说话人变化提示。
- 探索会议模式和照护者协助模式。
